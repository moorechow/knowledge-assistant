# 用户管理系统 — 第一步 + 第二步功能规划

> 目标：为知识库助手增加用户注册 / 登录 / 登出 / 密码修改功能，实现多用户隔离访问。

---

## 第一步：认证模块 `src/auth.py`

### 1.1 新增依赖

`requirements.txt` 追加一行：

```
bcrypt>=4.0
```

### 1.2 数据库初始化 `init_db()`

| 项 | 规格 |
|----|------|
| 存储引擎 | SQLite，文件路径 `data/users.db` |
| 首次调用行为 | 自动建表，创建默认 admin 账户 |
| 默认 admin | 用户名 `admin`，密码 `admin123`，首次登录强制修改 |

```sql
CREATE TABLE IF NOT EXISTS users (
    id            INTEGER PRIMARY KEY AUTOINCREMENT,
    username      TEXT NOT NULL UNIQUE,
    password_hash TEXT NOT NULL,
    email         TEXT,
    role          TEXT DEFAULT 'user',
    is_active     BOOLEAN DEFAULT 1,
    created_at    DATETIME DEFAULT CURRENT_TIMESTAMP,
    last_login    DATETIME
);
```

### 1.3 注册 `register_user(username, password, email?) → User`

| 校验项 | 规则 | 错误提示 |
|--------|------|---------|
| username 长度 | 3-20 字符 | "用户名需 3-20 个字符" |
| username 格式 | 仅字母、数字、下划线 | "用户名只能包含字母、数字、下划线" |
| username 唯一 | 查库去重 | "用户名已被占用" |
| password 长度 | 最少 6 字符 | "密码至少 6 个字符" |
| email 格式 | 如提供，需符合 x@x.x | "邮箱格式不正确" |
| 成功返回 | `User` 对象（不含 password_hash） | — |
| 失败返回 | `None` + 错误原因字符串 | — |

### 1.4 登录 `login_user(username, password) → User`

| 校验项 | 规则 | 错误提示 |
|--------|------|---------|
| 用户存在性 | 查库 | "用户名或密码错误"（不区分原因） |
| 账户状态 | `is_active == True` | "账户已被禁用" |
| 密码验证 | bcrypt.checkpw | "用户名或密码错误" |
| 成功 | 更新 `last_login`，返回 User | — |
| 失败 | 返回 None + 统一错误 | — |

### 1.5 登出 `logout_user()`

| 项 | 规格 |
|----|------|
| 行为 | 清除 Streamlit session_state 中所有用户相关字段 |
| 调用时机 | 主页右上角"登出"按钮或 session 过期 |

### 1.6 密码修改 `change_password(user_id, old_pw, new_pw) → bool`

| 校验项 | 规则 | 错误提示 |
|--------|------|---------|
| 旧密码正确 | bcrypt 对比 | "原密码错误" |
| 新密码长度 | ≥ 6 字符 | "新密码至少 6 个字符" |
| 新 ≠ 旧 | 不能与原密码相同 | "新密码不能与原密码相同" |
| 成功 | 更新 password_hash | 返回 True |

### 1.7 辅助函数

| 函数 | 签名 | 说明 |
|------|------|------|
| `get_user_by_id` | `(user_id) → User` | 按 ID 查用户 |
| `get_user_by_username` | `(username) → User` | 按用户名查用户 |
| `hash_password` | `(plain) → str` | bcrypt 哈希 |
| `verify_password` | `(plain, hashed) → bool` | bcrypt 验证 |
| `is_logged_in` | `() → bool` | 检查 session_state 是否有 user_id |
| `get_current_user` | `() → User` | 从 session_state 取当前用户 |
| `require_admin` | `() → bool` | 检查当前用户是否为 admin |
| `list_users` | `() → list[User]` | admin 用，列出所有用户 |
| `toggle_user_active` | `(user_id) → bool` | admin 禁用/启用用户 |
| `reset_password` | `(user_id, new_pw) → bool` | admin 强制重置密码 |

### 1.8 User 数据类

```python
@dataclass
class User:
    id: int
    username: str
    email: str | None
    role: str           # "admin" / "user"
    is_active: bool
    created_at: str
    last_login: str | None

    @property
    def is_admin(self) -> bool:
        return self.role == "admin"
```

### 1.9 错误码定义

| 错误码 | 含义 |
|--------|------|
| `USERNAME_TOO_SHORT` | 用户名太短 |
| `USERNAME_TOO_LONG` | 用户名太长 |
| `USERNAME_INVALID` | 用户名含非法字符 |
| `USERNAME_TAKEN` | 用户名已存在 |
| `PASSWORD_TOO_SHORT` | 密码太短 |
| `PASSWORD_MISMATCH` | 两次密码不一致 |
| `INVALID_CREDENTIALS` | 用户名或密码错误 |
| `ACCOUNT_DISABLED` | 账户已禁用 |
| `OLD_PASSWORD_WRONG` | 原密码错误 |
| `DB_ERROR` | 数据库异常 |

## 第二步：前端页面改造 `src/web.py`

### 2.1 页面路由

Streamlit 无内置路由，通过 `session_state.page` 切换：

```
┌────────────────────────────────────────────┐
│  st.session_state.page                     │
│                                            │
│  "login"   → 渲染登录页                     │
│  "register" → 渲染注册页                    │
│  "main"    → 渲染知识库助手主页              │
│                                            │
│  默认: 如果 is_logged_in() → "main"         │
│        否则 → "login"                      │
└────────────────────────────────────────────┘
```

### 2.2 登录页面

| 元素 | 规格 |
|------|------|
| 标题 | "知识库助手 — 登录" |
| 用户名输入框 | placeholder="请输入用户名" |
| 密码输入框 | type="password", placeholder="请输入密码" |
| 登录按钮 | 居中，主色调 |
| 注册链接 | "还没有账户？立即注册" → 切到 "register" |
| 错误提示 | 红色文字，显示具体原因（统一为"用户名或密码错误"） |
| 登录成功 | 存 session_state → 切到 "main" → rerun |
| 空字段校验 | 用户名或密码为空时，按钮 disabled 或前端校验 |

### 2.3 注册页面

| 元素 | 规格 |
|------|------|
| 标题 | "创建新账户" |
| 用户名输入框 | placeholder="3-20 位字母/数字/下划线" |
| 密码输入框 | type="password", placeholder="至少 6 位" |
| 确认密码 | type="password", placeholder="再次输入密码" |
| 邮箱输入框（可选） | placeholder="用于找回密码（选填）" |
| 注册按钮 | 居中，主色调 |
| 登录链接 | "已有账户？去登录" → 切到 "login" |
| 前端校验 | 用户名/密码/确认密码空则提示，密码长度 < 6 提示 |
| 两次密码不一致 | 红色提示 "两次输入的密码不一致" |
| 注册成功 | 自动登录 → 切到 "main" → "注册成功！" toast |

### 2.4 主页改造

| 改造点 | 规格 |
|--------|------|
| 移除全局 `@st.cache_resource` 单例 | `KnowledgeAssistant` 改为按 user_id 实例化 |
| 右上角信息栏 | `st.columns` 放右边：用户名 badge + "修改密码" + "登出" |
| 用户名显示 | `f"👤 {username}"` 或 `f"👤 {username} (管理员)"` role 不同样式 |
| 登出按钮 | 调用 `logout_user()` → 切到 "login" → rerun |
| 侧边栏底部 | 显示当前用户名和角色 |
| 统计面板 | 只显示当前用户的数据 |

### 2.5 修改密码弹窗/表单

| 元素 | 规格 |
|------|------|
| 入口 | 右上角 "修改密码" 链接或侧边栏入口 |
| 展示方式 | 侧边栏 expander 或 st.dialog |
| 旧密码 | type="password" |
| 新密码 | type="password", ≥ 6 字符校验 |
| 确认新密码 | type="password", 两次一致校验 |
| 提交按钮 | "确认修改" |
| 成功 | "密码修改成功" toast，关闭弹窗 |
| 失败 | 红色文字提示具体原因 |

### 2.6 页面样式统一

| 项 | 规格 |
|----|------|
| 登录/注册页面居中 | 卡片式布局，白色背景，圆角边框 |
| 按钮颜色 | 主色调蓝色 `#1f77b4` |
| 链接样式 | 蓝色下划线文字 |
| 错误提示 | 红色背景浅红色边框 `#fce4e4` |
| 成功提示 | 绿色 toast |

### 2.7 Session State 字段定义

| 字段 | 类型 | 说明 |
|------|------|------|
| `page` | `str` | 当前页面 "login"/"register"/"main" |
| `user_id` | `int` | 登录后存储 |
| `username` | `str` | 登录后存储 |
| `role` | `str` | "admin" 或 "user" |
| `messages` | `list` | 对话历史（已有，保留） |
| `pending_files` | `list` | 待上传文件（已有，保留） |
| `error` | `str` | 全局错误提示（可选） |
| `success` | `str` | 全局成功提示（可选） |

### 2.8 安全注意事项

| 项 | 措施 |
|----|------|
| 密码明文 | 绝不出现在日志、session_state 中 |
| SQL 注入 | 使用 SQLAlchemy 参数化查询 |
| Session 劫持 | Streamlit 自动管理，退出即失效 |
| 暴力破解 | 二期可加登录失败次数限制 + 冷却时间 |
| 密码传输 | 部署 HTTPS 后加密传输 |
